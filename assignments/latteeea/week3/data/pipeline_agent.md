- 지금의 structured field기반이 아닌 multi-agent로 가야하지 않나?
    
    근데 진짜 건강에 대한 신뢰성 있는 추천 시스템이 되려면 지금 같은 코드 기반 다중 필터링이 아니라
    agent가 여러개 결합돼서 서로 상태를 공유하고 검증하는게 자동화된걸로 가야할것같아. 예를 들어
    paper에서 mechanism에 대한 근거가 나왔으면 그 메커니즘을 학습해서 다른 rule들에서 추천하는 성분을 뽑아낼때 그 메커니즘에서 참고해야할게 있으면 알아서 참고하는 식으로. 이런 건 어때? 이런 서비스가 있나? 이런 서비스랑 지금 프로젝트 구조랑 비교해봐봐

    **논문 PDF 가 있으면**
    
    **→ 추출 에이전트 : EvidenceRow + 메커니즘 노드 → 지식 그래프
    → 집계 에이전트 : 근거 점수화 + 메커니즘 일관성 검증 
    → 추천 에이전트 : 후보 성분 생성 + “이 메커니즘에서 지지되는가” 자기 검증
    → 검증 에이전트 : 추천 이유가 저장된 메커니즘과 모순되지 않는지 교차 검증**
