## 🗂 오늘 작업

spotify API 지원 중지됨 → 배포된 앱에서 에러 및 기능 사용 불가 상태 → 다른 API로 갈아타기 

## ⚙ 기술 / 개념

🐛 트러블슈팅

- 문제: 음악 검색을 spotify api 를 사용해서 구현한 상태였는데 배포된 앱들에서 공통적으로 음악 검색이 안되는 문제 발생
- 원인: 403 에러가 뜨는데 우리쪽에서 spotify 관련 인증 처리하는 건 엑세스 키 및 클라이언트 ID 밖에 없었고 기존에 쓰던 건 무료 버전임을 알고 있었음.
    - 우리 코드 문제일리가 없음 → 배포된 앱에서 에러가 난 거기 때문에
    - 백엔드 문제일리도 없음 → 백엔드 코드를 바꾼 적이 없었고 DB 문제라기엔 다른 기능들은 잘 운영되고 있었음
    
    ⇒ spotify 문서 찾아보기
    
    → 바로 전날에 프리미엄 계정을 가진 클라이언트의 ID여야 API가 사용가능하도록 변경되었고 이게 2월부터 알려졌지만 우리는 메일 같은 걸 받지 못해서 모르는 상황이었던 것 
    
- 해결: 제목이나 가수 키워드를 검색해서 그와 관련된 노래를 검색하는 기능을 구현해야했는데 스포티파이 말고 해당 기능을 제공하는 다른 API를 찾아보았고 애플에서 제공하는 아이튠즈 API는 별도의 인증이나 제한 없이 사용 가능해서 이걸로 빨리 교체했다.

## 🧠 인사이트

내가 외부 API를 사용하고 있다면 그 업체에 대한 메일 수신 동의나 SNS를 팔로우 하고 있어야겠다.. 다른 프로젝트에서 아임웹 API도 사용하고 있는데 거기는 메일 수신 동의를 해놨는데 스포티파이는 안해놔서 차마 이 정보를 알지 못했다. 다음부터는 이를 예방하기 위해 팔로우를 잘 하고 있어야겠다..
